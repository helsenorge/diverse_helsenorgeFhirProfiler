# Helsenorge FHIR Profiles

This repository is a modern Windows-friendly FHIR R4 Implementation Guide project using FSH, SUSHI, Firely Terminal, GitHub Pull Requests, and Simplifier.

## Source, Generated, And Imported Files

`input/fsh/**` is the source of truth. Edit profiles, extensions, terminology, invariants, and examples there.

`fsh-generated/resources/*.json` is generated by SUSHI. These JSON files are committed so Simplifier can import stable generated resources from the `main` branch after Pull Request review and validation.

`output/**` is generated IG website output from the HL7 IG Publisher. It is not committed.

Simplifier should import only:

```text
fsh-generated/resources/*.json
```

from the `main` branch. Do not configure Simplifier to import `input/fsh/**`, `output/**`, or local cache folders.

## Folder Structure

```text
.
|-- input/
|   |-- fsh/
|   |   |-- codesystems/     # Source: CodeSystem definitions written in FSH
|   |   |-- examples/        # Source: example instances written in FSH
|   |   |-- extensions/      # Source: Extension definitions written in FSH
|   |   |-- instances/       # Source: supporting instances such as NamingSystem
|   |   |-- invariants/      # Source: reusable FHIRPath invariants
|   |   |-- profiles/        # Source: StructureDefinitions written in FSH
|   |   |-- valuesets/       # Source: ValueSet definitions written in FSH
|   |   `-- aliases.fsh      # Source: shared aliases
|   `-- pagecontent/         # Source: Markdown pages for the IG website
|-- fsh-generated/
|   |-- resources/           # Generated: JSON resources from SUSHI, committed for Simplifier
|   |-- data/                # Generated: SUSHI support data, not committed
|   |-- includes/            # Generated: SUSHI include files, not committed
|   `-- fsh-index.txt        # Generated: SUSHI index, not committed
|-- scripts/                 # Source: Windows PowerShell build and validation scripts
|-- .github/workflows/       # Source: GitHub Actions validation
|-- sushi-config.yaml        # Source: SUSHI and IG metadata
|-- ig.ini                   # Source: IG Publisher entry point
|-- package-list.json        # Source: package publication metadata
`-- package.json             # Source: Node tooling and project scripts
```

## Prerequisites

Install these tools on Windows:

- Node.js 20 or newer
- Java 17 or newer
- Firely Terminal
- VS Code

Then install Node dependencies:

```powershell
npm install
```

## Build Commands

Generate JSON resources with SUSHI:

```powershell
.\scripts\build-sushi.ps1
```

Automatically regenerate JSON whenever a `.fsh` source file changes:

```powershell
.\scripts\watch-sushi.ps1
```

or:

```powershell
npm run watch
```

Validate generated JSON with Firely Terminal:

```powershell
.\scripts\validate-firely.ps1
```

Validate only generated example resources with Firely Terminal:

```powershell
.\scripts\validate-examples-firely.ps1
```

or:

```powershell
npm run validate:examples
```

Run the local CI sequence:

```powershell
.\scripts\ci.ps1
```

IG website generation is not part of the required PR validation. Use the manual GitHub Actions workflow `Build IG Website` when a website artifact is needed. The current delivery flow is SUSHI JSON generation plus Firely validation, with Simplifier importing generated JSON from GitHub.

## Pull Request Flow

1. Edit FSH under `input/fsh/**`.
2. Run `.\scripts\build-sushi.ps1`.
3. Commit both changed FSH and changed JSON under `fsh-generated/resources`.
4. Open a Pull Request.
5. GitHub Actions runs SUSHI generation, checks generated JSON is current, and runs Firely validation.
6. Merge only after review and validation pass.
7. Simplifier imports generated JSON from the `main` branch.

## Versioning

The package version is controlled in `sushi-config.yaml`:

```yaml
version: 2.5.1-beta
releaseLabel: 2.5.1-beta
```

Update this when preparing a new Simplifier package version. The matching package publication metadata is in `package-list.json`.

Recommended release branch naming:

```text
release/2.5.1-beta
```

## Simplifier Setup

Configure Simplifier package publishing/import from GitHub using:

- Branch: `main`
- Include path: `fsh-generated/resources/*.json`
- Exclude source paths: `input/fsh/**`, `input/pagecontent/**`
- Exclude generated website output: `output/**`

This keeps Simplifier downstream of Pull Request review and Firely validation.
